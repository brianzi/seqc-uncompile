### A.1 — IF-116 `Expression::valueType` int → `EDirection` enum

- **Status today**: rename done (`valueType` → `direction`).
  Type-fix (int32_t → `enum class EDirection`) deferred.
- **Why deferred** (per IF-116 entry, commit `43b12c9`): `EDirection`
  is `enum class : int32_t` so strict typing forbids implicit int
  conversion. ~30 literal-int assignment sites in `expression.cpp`
  plus 12 sites in the autogenerated `seqc_parser.tab.c`
  (regenerated from `seqc_parser.y`). Each site would become
  `EDirection::eIN` / `EDirection::eINOUT` / etc.
- **Cost to actually do it**: low if we touch only `expression.cpp`
  + `seqc_parser.y` (regen tab.c). Realistic ~1–2 hours, including
  one build + diff cycle. The consumer site at `seqc_ast.cpp:43`
  already does `static_cast<EDirection>(e.direction)` so no churn
  there. Risk: if the parser-generated `tab.c` has subtle reads
  that don't look like assignments, we could miss one and get a
  build error — but that's discoverable in one `cmake --build .`.
- **Recommendation**: **do-now if Phase Q is opened anyway** — it's
  a natural Phase Q candidate (cosmetic + type-safety polish, no
  behaviour change). Otherwise **defer-formally** — it has zero
  behavioural impact and the audit note already documents why.
- **Test impact**: none (compile-time-only refactor).

## Part C — Phase Q (226 cosmetic items) refinement

Phase Q is defined in `SYNTHESIS.md` §6 + §13 as "226 yes-low + unsure
rows; defer to future style/cosmetic pass". Sampling rows from
`01_types.md`, `05b_custom_functions_play.md`, `22_device_factories.md`
(and counts from all 54 batch reports) lets me categorize them.

### Bucket 1 — "Pure cosmetic, mechanical" (~60 items)

**Pattern**: snake_case→camelCase, `_` consistency, dead constants
that should be either removed or marked unused, locals named after
disassembly registers (`rbx`, `ecx`, etc).

**Examples** (sampled):
- `kChannelTag_I` → `kChannelTagI` (Cluster O has 4 like this — already approved)
- `mergeWaveforms::funDescr2` (local; duplicates outer `funDescr`)
- `setSinePhase::resultPtr`/`rbx` (disasm-leakage locals — also in §7 dead code)
- `playIndexed::regInv` → `regInvalid`
- `play::reg0` / `regInv` → `regZero` / `regInvalid`

**Strategy**: **single sweep commit per cluster of related items**
(e.g. one commit "remove disasm-leakage local names across batches
05c/05d/19a", one commit "snake_case → camelCase polish", one
commit "rename `regInv` → `regInvalid` consistently").

**Estimated effort**: 1 day total. Build + test cycle per commit.
Zero behavioural risk. Net **token / future-grep** improvement is
real.

### Bucket 2 — "Borderline naming preference" (~80 items)

**Pattern**: Audit flagged with two plausible options
(`keep current` vs `something more descriptive`) and didn't pick.
Often local variables in `mergeWaveforms`, `play::*`, `writeToNode`,
`setWaitCyclesReg`. Frequently the better name is *known* (audit
proposed it) but renaming it anywhere requires deciding whether
the surrounding sibling locals should also rename in lockstep
(coordinated-rename tag).

**Examples**:
- `mergeWaveforms::useFunDescrPath` (path-A vs path-B selector)
- `play::numChannels` → `numChannelGroups` (semantic mismatch)
- `play::channelIndex` → `deviceIndex`
- `play::mask` → `triggerMask`
- `writeToNode::d/d2/d3/dHi/dF2` (shadowed nondescript names)

**Strategy**: **case-by-case decision, batched per-function**.
Best done by reading the full function once and renaming all locals
together. ~10 functions to revisit, ~30 min each.

**Estimated effort**: 1–2 days. Risk: micro — could mask a real
shadowing bug, but a build+test catches it.

### Bucket 3 — "Already-resolved during Phase D/R but never reconciled" (~30 items)

**Pattern**: Items that audit listed as low-conf in §6 but a Phase
D/R commit incidentally renamed/removed/clarified them. The audit
note hasn't been crossed off. Easy wins: confirm gone → strike
through.

**Examples**:
- `Cache::appendMode_` (low-conf in §6) — fixed by Cluster C in Phase D
- `useAbsolute` polarity (§6 in batches 39/47/52) — fixed by IF-117
- `Resources::parentWeak_` (low-conf) — fixed by Cluster N (IF-122)
- `DeviceOpts::*` constants — namespace removed (IF-121, batch 22)
- Several `kSuser*` constants flagged as "declared, never referenced"
  in §6 — wired up by IF-120

**Strategy**: One reconciliation pass — open SYNTHESIS.md §6, walk
each row, mark `(closed by Phase D commit X)` or `(closed by IF-Y)`,
remove from open count. Output: a reduced §6 of ~196 items rather
than 226.

**Estimated effort**: 2 hours. Pure note-keeping; no source change.
**This is the highest-ROI single action.** Without it, future Phase
Q work re-investigates already-resolved items.

### Bucket 4 — "Genuinely low-value, leave alone / formally close" (~56 items)

**Pattern**: Either (a) the symbol is a `kSuser*` / `kDev*` constant
that is declared-but-never-referenced and the audit's only proposal
was "remove if dead", but the constant is part of an authoritative
hardware-address map worth keeping for documentation; or (b) the
proposed rename is genuinely a coin-flip with no semantic edge
(e.g. `useYSuffix` vs `interleaveSecondary` — Arb 7 was already
resolved as `useYSuffix` kept).

**Examples**:
- `zhinst::kSuserUserRegBase` / `kSuserRTLoggerReset` / `kSuserQAResultLen`
  / `kSuserSweepStartLo/Hi` (unreferenced but documented hardware addrs — keep)
- `zhinst::kRateInherit` / `kNoWaveIndex` / `kNoNodeId` / `kNoPlayIndex`
  (sentinel constants — keep)
- `zhinst::kDevPreSHFLI` (only one use site, name describes ordering)
- `zhinst::kSuserNodeFreqCommit` (comment-only; one use site)
- `setWaitCyclesReg::res` (unused param — could remove but it
  matches the binary signature)
- `mergeWaveforms::useYSuffix` (already adjudicated by Arb 7 → keep)
- `DeviceOpts::SubtypeMask`/`Subtype1..4`/`FF`/`RTR`/`PLUS`/`LRT`
  (unused; meaning unverified — close as wontfix; documentation only)

**Strategy**: One audit-folder commit — add a "Phase Q closed:
intentionally kept" sub-section to SYNTHESIS.md §6 listing these
items. No source change. Closes ~56 items formally.

**Estimated effort**: 2 hours.
