# AWG Device Properties {#notes_awg_device_props}

`AwgDeviceProps` is the per-device descriptor consumed by the
compiler back-end: ELF size limits, the waveform-memory base
(`addressImpl`), the on-wire sample format, the device-family flag
(`isHirzel`), and the LabOne node-tree path patterns used to push
the compiled ELF down to the instrument.

Headers / sources:

- `reconstructed/include/zhinst/device/awg_device_props.hpp`
- `reconstructed/src/device/awg_device_props.cpp`

For the wider device taxonomy and the `AwgDeviceType` enum see
the device-type notes in the source headers.  For per-device sequencer
/ memory limits see \ref notes_device_constants.

## Fields

| Field                  | Type            | Meaning                                                   |
|------------------------|-----------------|-----------------------------------------------------------|
| `deviceType`           | `AwgDeviceType` | One-hot bit identifying the device family.                |
| `elfDataPattern`       | `std::string`   | Node path the compiled ELF is written to.                 |
| `elfProgressPattern`   | `std::string`   | Node path reporting upload progress.                      |
| `enablePattern`        | `std::string`   | Node path that arms / runs the sequencer.                 |
| `maxElfSize`           | `uint64_t`      | Maximum ELF size accepted by the device firmware.         |
| `addressImpl`          | `uint32_t`      | Waveform-memory base / sequencer entry-point address.     |
| `sampleFormat`         | `SampleFormat`  | On-wire sample encoding (0 = UHF, 1 = HDAWG/SG, 2 = QA).  |
| `isHirzel`             | `bool`          | Hirzel-generation device — selects ISA back-end.          |
| `fpgaRevisionPattern`  | `std::string`   | Node path queried for FPGA / slave revision.              |

## Per-device values

| Device   | `isHirzel` | `maxElfSize`         | `addressImpl` | `sampleFormat` | Path patterns | Revision path |
|----------|:----------:|----------------------|--------------:|:--------------:|---------------|---------------|
| UHFLI    | 0 | `0x10000000` (256 MiB)         | `0xd0000000` | 0 | Default       | `fpgarevision`   |
| UHFQA    | 0 | `0x10000000`                   | `0xd0000000` | 0 | Default       | `fpgarevision`   |
| HDAWG    | 1 | `0x80000000` if `ME` option set, else `0x10000000` | `0x80000000` | 1 | Default       | `slaverevision`  |
| SHFQA    | 0 | `0x80000000` (2 GiB)           | `0x00000000` | 2 | GrimselQa     | `fpgarevision`   |
| SHFSG    | 1 | `0x80000000`                   | `0x00000000` | 1 | GrimselSg     | `fpgarevision`   |
| SHFQC_SG | 1 | `0x80000000`                   | `0x00000000` | 1 | GrimselSg     | `fpgarevision`   |
| SHFLI    | 1 | `0x80000000`                   | `0x00000000` | 1 | GrimselLi     | `fpgarevision`   |
| GHFLI    | 1 | `0x80000000`                   | `0x00000000` | 1 | GrimselLi (alias `Gurnigel`) | `fpgarevision` |
| VHFLI    | 1 | `0x80000000`                   | `0x00000000` | 1 | GrimselLi (alias `Maloja`)   | `fpgarevision` |

`isHirzel` selects between the Cervino and Hirzel assembler
back-ends — see \ref notes_cervino_vs_hirzel.

HDAWG's `maxElfSize` is the only field that depends on the runtime
device-options query (`ME` — Memory Extension).  Every other field
is a compile-time constant of the device family.

## Path-pattern groups

`elfDataPattern` / `elfProgressPattern` / `enablePattern` are
generated by substituting `$device$` (instrument serial) and
`$index$` (AWG / channel index) into one of four template sets:

| Group     | Data                                              | Progress                                              | Enable                                       |
|-----------|---------------------------------------------------|-------------------------------------------------------|----------------------------------------------|
| Default   | `/$device$/awgs/$index$/elf/data`                 | `/$device$/awgs/$index$/elf/progress`                 | `/$device$/awgs/$index$/enable`              |
| GrimselQa | `/$device$/qachannels/$index$/generator/elf/data` | `/$device$/qachannels/$index$/generator/elf/progress` | `/$device$/qachannels/$index$/generator/enable` |
| GrimselSg | `/$device$/sgchannels/$index$/awg/elf/data`       | `/$device$/sgchannels/$index$/awg/elf/progress`       | `/$device$/sgchannels/$index$/awg/enable`    |
| GrimselLi | `/$device$/raw/awgs/$index$/elf/data`             | `/$device$/raw/awgs/$index$/elf/progress`             | `/$device$/raw/awgs/$index$/enable`          |

`fpgaRevisionPattern` is `"/$device$/system/fpgarevision"` for every
device except HDAWG, which uses `"/$device$/system/slaverevision"`.

## Error path

`makeUnsupportedAwgSequencerErrorMessage(deviceType, sequencer)`
produces:

```
Unsupported device or sequencer type (<dtype>, sequencer: <seq>).
```

Sequencer names are `"auto"` (0), `"QA"` (1), `"SG"` (2), or
`"unknown"` for any other value.
